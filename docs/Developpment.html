## Development

### Contributing

  • Coding Standards: Please follow our coding standards when contributing to the project. Refer to the [CONTRIBUTING.md](CONTRIBUTING.md) file for the coding style guide.

  • Code of Conduct: We adhere to a code of conduct to ensure a friendly and inclusive community. Review our [Code of Conduct](CODE_OF_CONDUCT.md) before contributing.

  • Submitting Contributions: To contribute to this project, follow these steps:

    1. Fork the repository.

    2. Create a new branch for your feature or bug fix.

    3. Make your changes, add tests if applicable, and ensure all tests pass.

    4. Create a pull request, and our team will review it.

### Building

  • Prerequisites: Ensure you have the following prerequisites installed:

    • C++ compiler (g++)

    • CMake (version 3.0 or higher)

    • SFML library (version 2.5.1)

  • Build Instructions: Follow these steps to build the game from the source code:

    `bash mkdir build cd build cmake .. make `

  • Configuration: If necessary, configure your build by setting environment variables or options. For example, you can use -DCMAKE_BUILD_TYPE=Release for a release build.

  • Troubleshooting: If you encounter build issues, consult our [Troubleshooting Guide](TROUBLESHOOTING.md) for solutions to common problems.

### Debugging

  • Debugging Tips: Here are some debugging tips for working on the project:

    • Use GDB or a similar debugger for C++ to step through your code.

    • Print debugging information using std::cout or custom log functions.

    • Set breakpoints in your IDE for interactive debugging.

  • Common Issues: If you come across common issues or errors, consult our [Common Issues](TROUBLESHOOTING.md#common-issues) section in the troubleshooting guide for solutions.

## API Reference

### Game Engine

  • AComponent:

    • This is an abstract base class for components in the ECS. It provides common functionality for derived component classes.

    • Key Elements:

      • getType(): Returns the type of the component.

      • setType(newType): Sets the type of the component.

    • Derived Classes: Health, Position, Speed, Body.

  • Health:

    • Represents the health component for entities.

    • Key Elements:

      • _health: Stores the current health value.

      • set_health(hp): Sets the health value.

      • get_health(): Gets the current health value.

    • Derived from: AComponent.

  • Position:

    • Represents the position component for entities.

    • Key Elements:

      • _x and _y: Store the X and Y coordinates of the position.

      • set_x(x) and set_y(y): Set X and Y coordinates.

      • get_x() and get_y(): Get X and Y coordinates.

      • get_position(): Get the position as a pair of X and Y coordinates.

    • Derived from: AComponent.

  • Speed:

    • Represents the speed component for entities.

    • Key Elements:

      • _speed: Stores the speed value.

      • set_speed(spd): Sets the speed value.

      • get_speed(): Gets the speed value.

    • Derived from: AComponent.

  • Body:

    • Represents the body component for entities with graphical representations.

    • Key Elements:

      • Rec_Sprite, P_texture, P_sprite, Name, clock: Store graphical information.

      • Methods to manage the graphical representation.

      • draw_playerWindow(window): Draws the entity in a game window.

    • Derived from: AComponent.

  • All_comps:

    • An empty class that doesn’t seem to have specific functionality provided in the code snippet.

  • Comps (Enum):

    • An enumeration defining various component types such as HEALTH, BODY, POSITION, etc.

  • Action (Enum):

    • An enumeration defining various actions, like Up, Down, Left, Right, Fire.

  • entitymanager:

    • A class to manage entities within the ECS.

    • Key Elements:

      • _entityl: A list of shared pointers to entities.

      • Methods for creating, adding, retrieving, and deleting entities.

    • Derived Classes: entity.

  • entity:

    • Represents an entity within the ECS.

    • Key Elements:

      • _id: Unique identifier for the entity.

      • _components: A list of shared pointers to components.

      • Template methods for adding, removing, and accessing components for the entity.

The CollisionSystem class is responsible for checking collisions between entities in the game. CollisionSystem collisionSystem; bool isCollision = collisionSystem.checkCollision(entityID1, entityID2);
